Sesión entre sitios (mismo navegador)

Con la sesión de Hecom ya activa en este navegador, los paneles se abren en el mismo sitio (p. ej. /credito, /pendientes) y el fragmento puede llevar tokens hacia el cliente de cada herramienta. Otro navegador, incógnito o perfil no comparte localStorage: ahí hace falta volver a entrar. Ningún redirect ni meta refresh del servidor debe consumir el # antes de que el cliente lea el hash. Importante: el handoff usa flujo implícito en el hash; el Supabase de la app del panel debe usar auth.flowType: 'implicit' (o detectar el hash y llamar setSession). Con pkce por defecto, Auth a veces borra la sesión al cargar.

Abrir panel usa una pestaña nueva y deja /destinos acá. Si bloquea popups, permití ventanas para hecom.club. La última elección se guarda solo en este navegador.
